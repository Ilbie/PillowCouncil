현재의 Ship Council은 정해진 순서대로(For-loop) 발언하고 반박하는 **'룰 기반 자동화(Rule-based Automation)'**에 가깝습니다. 이를 SOTA(State-of-the-Art) 급의 **'진정한 자율 에이전트(Autonomous Agentic System)'**로 끌어올리려면, 시스템이 스스로 사고하고 궤도를 수정하는 능력이 필요합니다.

현재 구조의 한계를 깨고 더 똑똑하고, 유연하며, 통찰력 있는 결과를 내기 위한 **4단계 심층 아키텍처 개선안**을 제안합니다.

---

### 🚀 Level 1. 정적 순서(For-Loop)에서 벗어난 '동적 라우팅(Dynamic Routing)'
현재 코드는 `for (const agent of preset.agents)`를 돌며 모든 에이전트가 무조건 한 번씩 의견을 내고, 또 무조건 한 번씩 반박합니다. 이는 인위적이고 낭비가 심합니다.

**💡 SOTA 개선안: Moderator 주도의 발언권 경매(Bidding) 및 지명 시스템**
*   **개념:** 실제 100분 토론처럼 모데레이터(사회자) 에이전트가 흐름을 읽고 **다음 발언자를 동적으로 지명**해야 합니다.
*   **구현 방향:**
    1.  라운드가 시작될 때, 모데레이터 LLM에게 현재 `DebateState`와 직전 발언을 줍니다.
    2.  모데레이터가 판단합니다: *"지금 A가 B를 강하게 비판했다. 다음 발언권은 방어할 수 있도록 B에게 주거나, 객관적 데이터를 제공할 C(데이터 분석가)에게 주어야 한다."*
    3.  모데레이터가 `{"nextSpeaker": "agent_B", "reason": "A의 비판에 대한 해명 요구"}` JSON을 반환합니다.
    4.  선택된 에이전트만 발언합니다.
*   **효과:** 무의미한 동어 반복이 사라지고, 논쟁이 치열한 지점에서 깊이 있는 티키타카(Tiki-taka)가 발생합니다.

### 🚀 Level 2. 메타인지(Metacognition)와 능동적 궤도 수정(Self-Correction)
현재 모데레이터는 단순히 내용을 요약하여 칠판(`DebateState`)에 적는 서기(Clerk) 역할만 합니다. 논의가 산으로 가도(에러 처리 논쟁 등) 막지 못합니다.

**💡 SOTA 개선안: '가드레일(Guardrail)'을 갖춘 능동적 개입(Intervention)**
*   **개념:** 토론 상태를 업데이트(`refreshDebateState`)할 때, **현재 논의가 원래 주제(User Prompt)와 얼마나 일치하는지 점검(Alignment Check)하는 로직**을 추가합니다.
*   **구현 방향:**
    1.  `debateStateSchema`에 `alignmentScore` (1~10)와 `deviationWarning` 필드를 추가합니다.
    2.  모데레이터 LLM이 요약할 때, 주제 이탈을 감지하면 `alignmentScore`를 낮게 매기고 경고문을 작성합니다.
    3.  점수가 특정 기준(예: 6점) 이하로 떨어지면, 오케스트레이션 엔진이 **시스템 개입 메시지(System Intervention Message)**를 타임라인에 강제 삽입합니다.
        *   *메시지 예:* "[System] 경고: 현재 패널은 핵심 비즈니스 가치에서 벗어나 지엽적인 기술 구현 세부사항에 매몰되어 있습니다. 즉시 원래 주제인 'OOO'의 비즈니스적 타당성 논의로 돌아오십시오."
*   **효과:** AI가 스스로의 토론 품질을 모니터링(Self-reflection)하여, 절대 논의가 우주로 가지 않게 붙잡습니다.

### 🚀 Level 3. 생각(Thinking)과 행동(Action)의 분리 (Plan-and-Solve)
현재는 `generateText` 한 번에 텍스트 생성과 도구(웹 검색, MCP) 사용이 동시에 일어납니다. 복잡한 경제 지표나 데이터를 논할 때 깊이가 얕아집니다.

**💡 SOTA 개선안: ReAct (Reasoning and Acting) 기반의 조사 페이즈 분리**
*   **개념:** 에이전트가 바로 입을 여는 것이 아니라, **"내가 발언하기 전에 무엇을 찾아봐야 할까?"**를 먼저 고민하고 검색한 뒤 발언하게 만듭니다.
*   **구현 방향:**
    1.  `streamMessageGeneration` 직전에 **Scratchpad(초안) 페이즈**를 추가합니다.
    2.  에이전트 LLM에게 *"너의 차례다. 발언하기 전에 필요한 데이터가 있다면 도구(웹 검색, MCP 등)를 써서 먼저 조사해라"*고 지시합니다.
    3.  에이전트가 `{"tool": "web_search", "query": "2026년 IMF 공식 개입 조건"}`을 반환하면 검색을 수행합니다.
    4.  검색 결과를 프롬프트에 주입(Grounding)한 후, 비로소 최종 발언(Opinion/Rebuttal) 텍스트를 생성합니다.
*   **효과:** "내 생각에는~" 식의 뇌피셜(환각, Hallucination) 토론이 아니라, **팩트와 최신 데이터에 기반한 극도로 전문적인 토론**이 가능해집니다.

### 🚀 Level 4. LLM-as-a-Compiler: 동적 페르소나 번역 (Dynamic Persona Adaptation)
현재 스크린샷의 가장 큰 문제는 'SaaS Founder'의 엔지니어/마케터 페르소나를 '거시경제(IMF)'라는 전혀 다른 도메인에 억지로 끼워 맞췄다는 점입니다.

**💡 SOTA 개선안: 프리셋 사전 컴파일링 (Pre-session Persona Compilation)**
*   **개념:** 세션을 시작(`run-session.ts`의 시작점)하기 전, **범용 LLM을 한 번 호출하여 주어진 주제(Prompt)에 맞게 프리셋의 역할을 '번역(Translate)'**합니다.
*   **구현 방향:**
    *   *주제:* "한국 IMF 가능성"
    *   *원본 페르소나:* "Staff Engineer (구조, 실패 조건, 유지보수, 아키텍처 담당)"
    *   *컴파일 페르소나 (AI가 자동 변환):* **"거시경제 시스템 분석가 (경제 구조의 취약점, 연쇄 부도 리스크, 재정 건전성 담당)"**
    *   이렇게 번역된 임시 페르소나들로 실제 토론을 진행합니다.
*   **효과:** 사용자가 어떤 엉뚱한 프리셋을 선택하든, 시스템이 알아서 해당 주제 도메인에 맞는 최고 전문가 패널로 변신(Role-playing adaptation)합니다. 진정한 의미의 자율 에이전트 구성입니다.

### 📝 결론: 어떤 것부터 적용해야 할까?

가장 적은 코드 수정으로 가장 압도적인(SOTA 급) 변화를 체감할 수 있는 순서는 다음과 같습니다.

1.  **(우선순위 1) Level 2. 메타인지 가드레일:** 모데레이터가 주제 이탈을 감지하면 시스템 메시지로 혼내는 기능. (단순 프롬프트 및 스키마 추가로 구현 가능)
2.  **(우선순위 2) Level 4. 동적 페르소나 번역:** 주제에 맞춰 에이전트의 직업 명세서를 한 번 세탁(Translation) 해주는 기능. (이것만 해도 IMF 주제에서 코딩 얘기하는 참사가 사라집니다.)
3.  **(우선순위 3) Level 1. 동적 라우팅:** 순서대로 말하는 방식에서 모데레이터 지명 방식으로 변경. (토론의 퀄리티와 몰입감이 완전히 달라집니다.)

이러한 요소들이 결합되면, Ship Council은 단순한 '챗봇 여러 개 띄워놓은 앱'을 넘어, **'주제를 스스로 이해하고, 방향을 통제하며, 전문가처럼 행동하는 완벽한 AI 의사결정 위원회'**로 진화할 것입니다.
